Introduction to CouchDB features in use in nEDM

The tutorials here introduce how the components of CouchDB are put to use in the nEDM-Interface. For more detailed information about the CouchDB API, see the documentation.

The nEDM interface uses the following functions (links are to external documentation):

  • view functions - query information available in documents in the database.
  • show functions - format documents to display e.g. html.
  • list functions - format results from a view.
  • update functions - update (or insert) new documents.
  • validate functions - provide security/limit types of changes to documents.
  • filter functions - filter results, e.g., for use in changes feeds.
  • changes feeds - get information on current changes to the database.

Information on how these are implemented are available in the child pages to this document. These functions are user defined in javascript and saved in design documents.
